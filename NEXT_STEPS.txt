Token Categories:

- SRC: Sources (api, db, file, mem, stream, queue, cache)
- RET: Return types (json, text, bin, stream, bool, num, list, dict)
- PARAM: Parameters (token, key, query, body, header, timeout, limit, times)
- CTL: Control flow (if, else, loop, break, continue, try, catch)
- TYPE: Data types (str, int, float, bool, list, dict, any)
- ERR: Error handling (retry, fail, log, ignore)
- MOD: Modifiers (async, batch, parallel, cached)

## What is a "Moltbot"?

A **moltbot** is an AI agent that uses MoltLang to communicate with other agents.

**Benefits:**
- 50-70% fewer tokens = lower API costs
- Semantic precision = fewer miscommunications
- Faster processing = better responsiveness
- More agents per context = larger fleets

**Example:**
```
# Traditional agent
agent_a.send("Please fetch the user data from the API and return it as JSON")
# 13 tokens, ambiguous

# Moltbot
agent_a.send("[OP:fetch][SRC:api][RET:json]")
# 3 tokens, unambiguous
```
